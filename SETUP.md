# 从零开始：在 VSCode 中搭建并运行本微信小程序项目

本文档假设你**完全没写过微信小程序**，按步骤操作即可。
你提到已经在 VSCode 配置好了相关 Key 和授权，下面的步骤里会指出哪里要用到它们。

---

## 第一步：安装必备工具

### 1.1 安装「微信开发者工具」（必须，这是官方唯一能预览/调试小程序的工具）

1. 打开浏览器访问：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 根据你的电脑系统（Windows / Mac）下载对应安装包并安装。
3. 安装完成后**打开一次**，用你的微信扫码登录（和登录公众号平台是同一套账号体系）。

> 注意：VSCode 本身**不能**运行/预览小程序，它只是用来"写代码"的编辑器。
> 真正的"运行、预览、点击体验"必须在「微信开发者工具」里完成。
> 你可以理解为：VSCode 写代码 → 微信开发者工具运行代码，两者配合使用。

### 1.2（可选，但推荐）VSCode 安装小程序代码高亮插件

在 VSCode 左侧插件市场搜索安装以下任一插件，让 `.wxml`/`.wxss` 文件有语法高亮，写代码体验更好：
- `minapp` （推荐，支持代码补全）
- 或 `WXML - Language Service`

---

## 第二步：把项目代码放到本地

如果你是从我这里下载到的项目文件（一个 zip 包或者一个文件夹）：

1. 解压到你想存放代码的位置，比如 `D:\projects\recycle-miniprogram`（Windows）或
   `~/projects/recycle-miniprogram`（Mac）。
2. 用 VSCode 打开这个文件夹：
   - 打开 VSCode → 菜单「文件 / File」→「打开文件夹 / Open Folder」→ 选中 `recycle-miniprogram` 文件夹。
3. 确认能在 VSCode 左侧文件树看到这样的结构（和 `docs/README.md` 里描述的一致）：
   ```
   app.js  app.json  app.wxss  project.config.json  sitemap.json
   config/  utils/  pages/  docs/
   ```

> 这一步在 VSCode 里**不需要执行任何命令**，纯前端原型阶段没有 npm 依赖、不需要 `npm install`。
> 整个项目就是普通的文本文件，打开文件夹即可编辑。

---

## 第三步：在「微信开发者工具」中导入并运行项目

1. 打开「微信开发者工具」。
2. 选择「小程序」项目类型 → 点击「+」新建/导入项目。
3. 在弹出的窗口中：
   - **目录**：选择第二步里的项目文件夹 `recycle-miniprogram`（注意选到这个文件夹本身，
     不要选到它的某个子文件夹）。
   - **AppID**：
     - 如果你已经有自己申请的小程序 AppID（在微信公众平台 mp.weixin.qq.com 注册小程序后获得），
       填入你自己的 AppID。
     - 如果还没有申请，点击下方「**使用测试号**」或保持项目里 `project.config.json`
       中预设的 `"appid": "touristappid"`（无 AppID 模式），同样可以正常预览原型，
       只是部分需要"线上"权限的接口（比如真实发消息）用不了，不影响你看页面和点交互。
   - 项目名称随意填，比如「资源回收原型」。
4. 点击「导入」/「确定」。
5. 稍等片刻，微信开发者工具中间会出现手机模拟器界面，看到「一键预约回收」首页即代表运行成功。

---

## 第四步：在模拟器里体验三端流程

### 4.1 体验"用户端"完整流程
1. 模拟器默认进入的就是用户端首页。
2. 点击「📷 拍照下单」「📞 一键呼叫」体验极简操作（模拟器里拨打电话会失败弹窗提示，真机才能真打）。
3. 点击地址栏 →「使用当前定位」自动填地址。
4. 选择时间 → 点击底部「立即预约」→ 会自动跳到「订单」Tab。
5. 点击底部「订单」Tab，切换顶部「待接单/待上门/已完成」筛选，体验状态展示。
6. 在"待接单"订单上点击【取消订单】，确认后订单会从列表消失（刷新小程序会恢复，这是正常的，因为是假数据）。
7. 点击底部「我的」Tab，查看地址、积分信息。

### 4.2 体验"商户端"完整流程
1. 在「我的」页面，点击最下方【🔄 我是回收商家，切换至商户端】。
2. 进入「接单大厅」，能看到 1 条待接单的订单（隐藏了手机号和门牌号，只显示小区名）。
3. 点击【抢单】，确认后这条订单从大厅消失。
4. 点击页面最下方【我的任务】，能看到一条"待上门"的任务（完整地址电话）。
5. 点击【确认完成】，在弹出的"记账"小窗里输入重量（比如 30），点击【确认发放并完成】，
   订单状态变为"已完成"并显示发放的积分。
6. 点击【一键导航】会调用地图（模拟器里大概率会提示失败，真机环境效果更好）。
7. 返回上一级，点击【推广中心】，查看专属推广码占位图和绑定客户数统计。
8. 点击【退出商户端】回到用户端首页。

---

## 第五步：修改代码后如何看到效果（开发的日常循环）

1. 在 VSCode 里修改任意 `.js` / `.wxml` / `.wxss` / `.json` 文件并保存。
2. 切回「微信开发者工具」窗口，它**默认会自动检测文件变化并重新编译**
   （如果没自动刷新，点击工具栏上的「编译」按钮，或快捷键 `Ctrl+B` / `Cmd+B`）。
3. 左侧模拟器界面会刷新显示最新效果。

> 调试技巧：点击微信开发者工具顶部的「调试器」，可以看到 Console 面板，
> 我在代码里写的 `console.log(...)` 都会打印在这里，方便你确认每次点击触发了什么逻辑。

## 第六步（后续）：何时需要申请正式 AppID

当你决定要把这个原型给"真实的、非开发者身份的普通人"在手机上体验时（不是在你自己电脑的模拟器里），
有两个选择：
1. **申请小程序 AppID 并发布体验版**：
   - 登录 https://mp.weixin.qq.com → 注册「小程序」账号 → 获得 AppID
   - 在「微信开发者工具」右上角点击「上传」，提交版本
   - 登录公众平台后台 → 「版本管理」→ 将刚上传的版本设为「体验版」
   - 在后台「成员管理」里添加体验者的微信号（个人开发者未发布前，只有加了"体验成员"的人能看到）
2. **暂时不想申请**，只想自己人在同一台电脑或局域网内看效果：
   可以使用微信开发者工具的「预览」功能，生成一个临时二维码，扫码后在真机微信里直接体验
   （这个不需要等审核，但同样需要你已经登录了开发者工具的微信账号，且对方微信号最好也加进体验成员，
   否则不同身份扫码可能受限，具体以届时微信平台规则为准）。

把这一步留到"原型体验满意，决定要继续往下做"的时候再处理即可，现阶段你自己在模拟器里点一遍已经足够验证交互合理性。
