# 版本管理文档（VERSIONS）

本文档汇总每个版本的改动内容，并给出对应的 git 提交命令和 tag 命令。
建议你每次按 `docs/VERSIONS.md` 里的"建议命令"操作，这样未来想回退到任意版本，
直接 `git checkout v1.x` 就能切回那个版本的代码状态，非常方便对比体验。

> 前提：你已经执行过 `git init`（如果还没有，先看 `docs/COMMANDS.md` 第2步）。

## v1.2 —— 照片位置/价格详情页/详细地址补充/未绑定商家皮肤（本次更新）

**内容**：

1. **照片位置调整**：拍照后的缩略图从"备注上方"移动到【一键呼叫/拍照下单】按钮下方，
   最多3张，拍完立即可见，避免用户不确定是否拍照成功。
   - 涉及：`pages/index/index.wxml` `.js` `.wxss`

2. **今日回收价"查看更多"**：首页只展示精选4项 + 【查看更多 ››】，
   新增 `pages/priceList` 页面，按"纸皮类/金属类/塑料类/家电类/衣物类"分类展示完整价格表
   （参照你提供的截图结构）。
   - 涉及：`pages/index`、新增 `pages/priceList`、`utils/mock.js`（`getPriceCategoryList`）

3. **地址详细门牌补充**：`wx.chooseLocation` 受系统/地图API限制，只能定位到"小区/楼栋附近"，
   无法像专业地图软件那样定位到"室内具体门牌"。所以改为：定位选点后弹出"补充门牌号"弹窗，
   用户手动输入楼栋/单元/门牌号，拼接成完整地址。同步应用到"亲友代下"页面。
   - 涉及：`pages/index`、`pages/familyBind`

4. **未绑定商家皮肤**：新增 `app.globalData.isBound` 全局状态：
   - `true`（已绑定）：显示专属商家横幅、显示一键呼叫、今日回收价显示商家自定义价
   - `false`（未绑定）：横幅变为"同城绿色回收大厅"、隐藏一键呼叫、拍照下单按钮拉满、显示城市参考价
   - 在"我的"页面新增【测试开关】，方便你来回切换体验两种状态（这是开发调试用的，真实上线后这个开关应该由后端根据真实绑定关系判断，不会让用户自己切）
   - 涉及：`app.js`、`pages/index`、`pages/mine`、`pages/priceList`

5. **订单评价（PRD V1.6 一并实现）**：
   - 待接单：保留【取消订单】红框按钮
   - 已完成 + 未评价：绿色【评价服务】按钮
   - 已完成 + 已评价：灰色文字"已评价 X星"
   - 点击评价服务：弹出星级(1-5) + 差评快捷标签(评分≤3星时显示) + 提交
   - 提交后：`mock.submitOrderComment()` 模拟"写库"（修改内存中订单的 `is_commented`/`rating`），
     页面局部刷新按钮状态，并提示"感谢评价，获得额外5积分！"
   - 涉及：`pages/orderList`、`utils/mock.js`（`submitOrderComment`，订单数据新增 `is_commented`/`rating` 字段）

```bash
git add .
git commit -m "feat(v1.2): 照片位置调整/价格详情分类页/地址门牌补充/未绑定商家皮肤/订单评价功能(V1.6)"
git tag -a v1.2 -m "照片展示位置+价格详情页+详细地址+商家绑定状态切换+订单评价"
```

## 后续版本命名建议

- 小的体验微调（不增加新页面/新数据结构）：`v1.2.1`、`v1.2.2`...（patch 版本号+1）
- 新增功能模块（比如真正做积分商城、做"为他预约"联动首页预填）：`v1.3`（minor 版本号+1）
- 接入真实后端、推翻 mock 数据结构：`v2.0`（major 版本号+1，标志"原型阶段结束，进入真实开发阶段"）

```bash
# 查看所有历史版本 tag
git tag

# 回退到某个历史版本查看（不会丢失当前代码，只是切换查看视角）
git checkout v1.1

# 看完想回到最新代码
git checkout main   # 如果你的主分支叫 master，就用 git checkout master
```

## 如果你想把 tag 也推送到远程仓库（GitHub/Gitee）

```bash
git push origin --tags
```
