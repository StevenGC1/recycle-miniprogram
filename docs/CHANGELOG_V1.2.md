# 更新日志 —— V1.2（含 V1.6 订单评价功能）

对应你这轮反馈的 4 个问题 + 订单评价 PRD。详细设计对照见 `docs/VERSIONS.md` 的 v1.2 章节，
这里只列【你需要重点验收的点】和【已知简化/限制】。

## 重点验收点

1. 首页拍照后，缩略图立刻出现在【一键呼叫/拍照下单】按钮下方，最多3张，超过3张时"+"号自动消失，点已有照片可放大预览，点右上角"×"可删除单张。
2. 首页"今日回收价"右上角新增【查看更多 ››】，点击进入 `pages/priceList`，按分类展示完整价目表。
3. 首页/亲友代下页面，点击地址栏 → 先走 `wx.chooseLocation` 定位选点 → 选完自动弹出"补充门牌信息"小窗，输入栋/单元/门牌号后拼接成完整地址。
4. "我的"页面新增一个绿色开关【是否绑定专属商家】，关掉后回首页看效果：横幅变"同城绿色回收大厅"，一键呼叫按钮消失，拍照下单按钮拉满整行。
5. 订单列表页，"已完成"且未评价的订单会出现绿色【评价服务】按钮，点击后弹出星级评价+差评标签弹窗，提交后按钮变成灰色"已评价 X星"文字，并弹出"获得额外5积分"提示。

## 已知简化/限制（不是bug）

- `wx.chooseLocation` 受限于地图API本身的精度，做不到自动识别到"xx大厦A栋"这种室内级别，这是任何小程序用这个官方接口都会遇到的限制，所以采用"定位大致位置+手动补门牌号"的折中方案。如果你希望体验更接近"高精度地图软件"，需要接入专门的地图SDK做POI搜索，这个工作量较大，建议放到正式做后端阶段再评估。
- "未绑定商家"开关是我加的**调试用工具**，方便你一个人切换两种状态看效果。真实上线后，这个状态应该由后端根据用户是否扫过商家码自动判断，不会让用户自己手动切。
- 评价提交后的"积分翻倍"逻辑，原型阶段只是固定弹出"获得额外5积分"的提示文案，没有真的去改"我的"页面里的积分数字（那个积分数字是另一份独立的mock数据），如果你需要这两处联动同步，告诉我，下一版本可以加上。
