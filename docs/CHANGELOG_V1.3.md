# 更新日志 —— v1.3（4个bug修复 + 商户端 V1.9 重构）

## Part A：4个bug修复

1. **地图选点点"取消"误弹补充门牌窗**：现在判断 `err.errMsg` 是否包含 `cancel`，是则什么都不做，不会弹窗。涉及 `pages/index/index.js`、`pages/familyBind/familyBind.js`。
2. **上门时间未按当前时间限制**：新增 `refreshPeriodOptions()`，超过中午12点自动禁用"今天+上午"，并在选中"今天上午"时已过点自动帮你切到"今天下午"，禁用项点击会提示"当前时间已过"。涉及 `pages/index/index.js/.wxml/.wxss`。
3. **定位逆地理编码固定写死南山区**：改成按经纬度范围粗略匹配深圳南山/福田、广州天河、北京朝阳、上海浦东几个城市，模拟器里改坐标会跟着变。涉及 `utils/mock.js`（`CITY_BOUNDS`/`mockReverseGeocode`）。
   - 同时澄清：**已绑定商家时不会定位**，这是设计本意（商家自己的价格不需要看城市），未绑定才会触发定位流程。
4. **客服微信号不能复制**：原来用的是 `wx.showModal` 原生弹窗（不支持自定义按钮），改成自定义弹窗 + `wx.setClipboardData` 复制。涉及 `pages/mine/mine.wxml/.js/.wxss`。

## Part B：商户端 V1.9 重构

### 身份判定与自动分流
- `utils/mock.js` 新增 `MERCHANT_APPLICATION`（审核状态：`none`/`pending`/`approved`）
- `app.js` 的 `onLaunch` 新增判定逻辑：审核通过的商户启动小程序会被 `wx.reLaunch` 直接拉进商户工作台，不需要每次手动切换
- 新增页面 `pages/merchantApply/merchantApply`：填写店铺名称/手机号/服务片区，提交后状态变 `pending`
- 用户端"我的"页新增入口【📋 申请入驻成为回收商】

### 商户端三大路由页面（取代原 hall/tasks/promotion）
1. **`pages/merchant/index`（工作台）**：合并接单大厅+我的任务，任务区分 [待上门]/[已完成]/[已取消] 三个子Tab；已完成卡片内直接回显用户评价星级和差评标签（不再做独立评价Tab）
2. **`pages/merchant/shop`（商户货架）**：【+发布二手闲置】（图片1-3张+名称+标价+库存），商品列表支持【下架】/【标记已售】；这里上架的商品是用户端"二手商城"的数据源（`merchantId` 自动关联）
3. **`pages/merchant/mine`（商户中心）**：数据看板（积分/今日成单数/今日回收量）+ 工具箱（我的地推码、自定义报价）

### 自定义报价联动
- 新增 `pages/merchant/priceConfig`：商户可编辑各品类价格，保存后通过 `mock.updateMerchantPrice()` 同步更新到居民端首页"今日回收价"和价格详情页（因为是同一份内存数据的引用，不需要额外同步逻辑）

### 测试专用入口（真实上线后会移除）
- "我的"页【🔑 强制切换为已审核商户，进入工作台】：跳过申请审核流程，直接体验商户端
- 商户中心【🔁 退出商户身份，返回用户端】：测试时来回切换方便

## 已知简化点（不是bug）

- "抢单"后订单的 `merchantId` 没有真的在 mock 数据里改成当前商户（只是从大厅列表移除），所以抢单后不会立刻出现在"待上门"里，这块涉及真实的"数据库写入"逻辑，留到接后端时处理。
- 商户审核从 `pending` 到 `approved` 在真实项目里由你在后台手动操作，原型阶段没有管理后台，所以放了【强制切换】按钮代替这一步。
- 自定义报价目前只能编辑你已经有的品类，新增/删除品类的功能没有做，按你的反馈再决定要不要加。
