# 常见问题排查指南（TROUBLESHOOTING）

## 问题1：真机调试报错 "file not found"，明明文件存在

**现象**：`错误码800059`、`pages/xxx/xxx.js, file not found`，但你打开文件夹确认文件确实存在。

**原因**：这是微信开发者工具"真机调试"模式的**缓存机制**问题。真机调试需要把代码包传到手机上运行，
工具有时不会自动检测到"新增的页面文件"，尤其是刚加了一个全新的页面（比如这次新增的 `pages/priceList`）。

**解决步骤（按顺序尝试）**：
1. 先退出真机调试（手机上断开/工具里点"结束调试"）。
2. 在微信开发者工具菜单栏：**工具 → 清除缓存 → 全部清除**。
3. 点击工具栏的【编译】按钮，确认普通的"模拟器"预览里页面是正常的（先排除是不是代码本身的问题）。
4. 重新点击【真机调试】，**等待二维码生成后用手机微信扫码重新进入**，不要用手机里之前缓存的旧版本。
5. 如果还报错，检查手机微信版本是否过旧（设置-关于微信-检查更新），过旧版本的基础库可能不支持某些新页面结构。

> 经验：每次新增"全新页面文件"（不是改已有页面的内容，而是新增 .wxml/.js/.json/.wxss 一整套）之后，
> 第一次真机调试前都建议养成习惯先做一次"清除缓存"，能避免大部分这类报错。

## 问题2：点击"选择地址"，直接弹出"模拟定位"，没有真的定位

**现象**：点地址栏后，没看到地图选点界面，直接弹出"补充门牌信息"弹窗，地址写着"模拟定位"。

**原因**：`wx.chooseLocation` 在**模拟器里默认没有设置位置信息**，导致接口直接报错走了我代码里的兜底分支
（这是我故意写的保护机制，避免没配置定位时整个流程卡死，但效果上看起来就是"没真的定位"）。

**解决步骤——让模拟器里的定位生效**：
1. 打开微信开发者工具的模拟器界面（不是真机）。
2. 模拟器顶部工具栏一般有个"更多"按钮（或者类似指南针/定位图标），点击后会有"模拟位置"的设置项。
3. 在地图上选一个点（比如随便点深圳市区某处），保存。
4. 重新回到小程序页面，再点击"选择地址"，这次应该能看到真实的地图选点界面（不再是兜底文案）。

**真机上的解决步骤**：
1. 真机第一次点击"选择地址"时，微信会弹出授权框"是否允许访问你的地理位置"，**必须点"允许"**。
2. 如果之前手滑点了"拒绝"，需要去手机：**微信 → 我 → 设置 → 隐私 → 小程序授权管理 → 找到这个小程序 → 打开"位置信息"**。
3. 同时确认手机系统层面的定位服务（GPS/位置服务）是打开的。

> 我已经在 `app.json` 里补充了 `permission` 字段声明位置权限用途，这是触发授权弹窗的必要配置，之前漏了。

## 问题3：定位精度只能到"小区附近"，到不了"xx大厦A栋"

这是 `wx.chooseLocation` 这个**官方接口本身的精度限制**，跟代码写法没关系——所有小程序用这个接口都一样，
做不到专业地图软件（比如美团外卖、滴滴）那种"室内级别"定位，那些软件背后通常接了更专业的地图SDK + POI搜索能力。

我们目前的解决方案是"折中方案"：定位到大致位置后，**弹窗让用户自己手动补一句门牌号**，
这是大部分轻量级小程序的常规做法，能用、体验也不差，只是确实做不到自动识别到"A栋"那种精度。
如果后续你觉得这个精度不够用，需要单独申请地图Key、接入POI搜索接口，这部分工作量比较大，
建议放到正式做后端阶段再评估是否需要投入。
